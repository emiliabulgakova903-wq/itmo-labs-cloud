1. Цель работы
Ознакомиться с основными понятиями и принципами работы объектного облачного хранилища, изучить функционал создания бакетов, управления файлами и настройки публичного доступа к данным.

2. Ход работы
2.1. Создание Cloud Storage Bucket
В рамках существующего проекта был создан бакет (bucket) для хранения объектов.

Имя бакета: bucket-lab3-ebulgakova.

Регион: Выбран стандартный регион (us-central1).

Настройка доступа: При создании был отключен параметр «Enforce public access prevention», что необходимо для последующей настройки публичного доступа к файлам.


2.2. Загрузка и организация файлов
В созданный бакет было загружено 4 изображения.

Для организации структуры данных была создана папкa.

Все загруженные файлы были перемещены в созданную папку через интерфейс консоли.

![Создание сервисного аккаунта](screenshots/ф2.png)

![Создание сервисного аккаунта](screenshots/ф3.png)

2.3. Настройка публичного доступа
Для реализации возможности просмотра файлов через интернет была изменена политика доступа:

В разделе Permissions был добавлен новый субъект (Principal): allUsers.

Назначена роль: Storage Object Viewer.

Это позволило сделать файлы доступными для чтения любому пользователю по прямой ссылке.

![Создание сервисного аккаунта](screenshots/ф4.png)

2.4. Тестирование ссылок
Через контекстное меню файлов были получены прямые ссылки (Public URL). При вставке ссылки в браузер изображения открываются корректно, что подтверждает правильность настройки прав доступа.

3. Завершение работы
После фиксации результатов все созданные ресурсы (папки, файлы и сам бакет) были удалены для предотвращения нецелевого использования облачных ресурсов.

4. Вывод
В ходе лабораторной работы были изучены принципы объектного хранения данных в Cloud Storage. Были освоены навыки создания хранилищ, структурирования данных с помощью папок и управления правами доступа, что является критически важным для обеспечения безопасности и доступности данных в облаке.
